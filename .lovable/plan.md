

# Улучшение UI курса "Основы управления изменениями"

## Текущие проблемы

1. **Курс открывается в модальном окне (Dialog)** -- тесное пространство, неудобно читать длинные тексты, нет ощущения полноценного обучения
2. **Карточки занятий и содержимое урока отображаются одновременно** -- перегруженный интерфейс, пользователь видит всё сразу
3. **Мультимедиа показаны как карточки со ссылками "Открыть ресурс"** -- нет встроенных плееров, контент не воспроизводится на месте
4. **Нет боковой навигации** -- невозможно быстро переключаться между занятиями
5. **Квиз и занятия на одном экране** -- скроллинг вниз для доступа к квизу
6. **Нет визуального прогресс-бара по занятиям** в шапке курса
7. **Лонгрид подан как обычные параграфы** -- без визуального выделения и структурирования

## План изменений

### 1. Полноэкранная страница вместо модального окна

Курс "Основы управления изменениями" (lm1) будет открываться как отдельная страница `/learning/course/lm1` (аналогично курсу lm3 на `CoursePage.tsx`), а не в Dialog.

### 2. Боковая панель навигации по занятиям

Левая колонка (фиксированная, ~280px) со списком занятий:
- Номер, название, длительность
- Иконка замка/галочки для статуса
- Подсветка активного занятия
- Кнопка "Квест" внизу

### 3. Улучшенная основная область контента

Правая колонка с содержанием текущего занятия:
- Шапка с заголовком, целью занятия и временем
- Блок "Лонгрид" -- стилизованные параграфы с отступами и выделением первого абзаца
- Встроенные мультимедиа-плееры: video-заглушки и audio-элементы прямо на странице
- Инфографика как стилизованные карточки (аналогично CoursePage)
- Чек-лист с интерактивными чекбоксами
- Кнопки "Завершить занятие" и навигация "Назад/Далее"

### 4. Прогресс-бар в шапке

Горизонтальная полоса прогресса с процентами и счетчиком пройденных занятий.

### 5. Улучшенный квиз

- Визуальная обратная связь после ответа (зеленый/красный)
- Пояснение к каждому вопросу после отправки
- Результат с рекомендацией

## Технический план

- Изменить обработчик кнопки "Открыть" для `lm1`: вместо `openStructuredCourse()` выполнять `navigate('/learning/course/lm1')`
- Убрать Dialog-логику для курсов, у которых есть полноэкранная страница

### Файл: `src/pages/CourseBasicsPage.tsx` (новый файл)

Создать новую страницу для курса "Основы управления изменениями" по образцу `CoursePage.tsx`, но с улучшениями:
- Двухколоночный layout: sidebar + контент
- Встроенные video/audio плееры вместо ссылок
- Стилизованные лонгриды с выделением ключевых мыслей
- Интерактивные чек-листы
- Плавная навигация между уроками

### Файл: `src/App.tsx`

- Добавить маршрут `/learning/course/lm1` для `CourseBasicsPage`

### Структура нового layout

```text
+----------------------------------------------------------+
|  <- Назад    Основы управления изменениями    ~220 мин   |
|  [============================== 60% ==================] |
+------------------+---------------------------------------+
| Занятие 1   [v]  |  Занятие 1. Природа изменений         |
| Занятие 2   [v]  |  Цель: ...                            |
| Занятие 3   [>]  |                                       |
| Занятие 4   [x]  |  [Лонгрид текст с выделениями]        |
| Занятие 5   [x]  |                                       |
|                  |  [Видео плеер]                         |
| Квест       [x]  |  [Аудио плеер]                        |
|                  |  [Инфографика - карточки]              |
|                  |                                       |
|                  |  Практические шаги:                    |
|                  |  [x] Шаг 1                             |
|                  |  [ ] Шаг 2                             |
|                  |                                       |
|                  |  [Завершить]  [<- Назад] [Далее ->]   |
+------------------+---------------------------------------+
```

Данные курса (5 занятий, мультимедиа, квиз) будут взяты из существующего объекта `structuredCourses['lm1']` в `LearningPage.tsx` и перенесены в новый файл с расширенным контентом для лонгридов.
